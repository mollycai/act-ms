## 第一次作业: 运营平台活动管理页面开发  
使用 React 或 Vue 框架，结合 Semi-UI 或 shadcn/ui 等组件库，开发一个运营活动管理页面（PC）。页面需包含活动首页、活动列表页与活动详情页，并满足以下具体要求：
- 活动首页  
  - 需展示平台当前进行中的重点活动，可包含 Banner、活动分类入口、公告区域等模块。  
  - 至少实现两种可切换的布局样式（如宫格/轮播样式）。  
  - 活动数据需从模拟接口获取，接口结构需设计合理。  
  - 页面需要预留组件化扩展空间，例如可按活动类型插入不同卡片组件。
- 活动列表页  
  - 支持按活动状态、时间范围、关键词搜索等筛选方式，筛选参数需保持在 URL 中。  
  - 列表支持分页或无限滚动，需结合接口分页实现。  
  - 每个活动列表项需包含活动标题、时间、状态、创建人等基础信息。  
  - 点击列表项可进入详情页。
- 活动详情页  
  - 展示活动基础信息、活动规则、活动参与数据（可使用模拟数据）。  
  - 支持编辑模式和只读模式，编辑提交后更新并反馈结果（可使用模拟数据）。  

### 考核标准
- 页面模块完整度与功能正确性，UI 交互体验，组件库使用的恰当程度。
- 模拟接口数据格式实现的合理性，组件化、可扩展结构设计的清晰度。
- 可使用 AI 编程，但需提交详细 AI Coding 使用报告，包括提示词、生成步骤、人工校验说明，不得纯粹的 Vibe Coding 作为最终结果。
  
### 加分项  
- 首页增加具有节奏感或过渡效果的 CSS 动画，可与活动主题联动，如节日主题动画。  
- 列表页增加体验优化：骨架屏、虚拟滚动、大数据下不卡顿、筛选交互的 Loading 状态处理等。  

## 第二次作业：通用页面引导功能设计  
在第一次作业基础上，为平台页面设计一个通用的新手引导系统，需可复用于多个页面，具体要求：
- 支持以“步骤”形式展示引导，例如为活动首页、列表页、详情页编写可配置的引导流程。  
- 引导逻辑需与页面业务解耦，通过 JSON 配置或注册机制统一管理。  

### 考核标准
- 禁止使用 AI 编程，独立完成，需提交完整设计文档，包括引导系统架构、模块划分、关键难点方案。
- 引导系统的可复用性、配置化程度。与现有页面的整合度及实际可用性。
- 系统设计文档的清晰度（架构图、流程图、关键接口说明）。

### 加分项
- 系统需考虑不同用户的数据差异，例如列表为空、用户权限不同导致某些按钮不可见等情况。  
- 支持对页面中的具体元素进行定位（如按钮、表单项、标签等），并允许为每个步骤编写提示文案与交互方式。  
- 当页面结构变化时，引导系统的适配成本越低越好。可从两方面入手：
  - 通过数据结构优化或 引入 AI 能力，提升引导的鲁棒性，小改动可自动兼容。
  - 提供可视化调试模式，让开发者能直观看到每个步骤定位的元素与顺序。
